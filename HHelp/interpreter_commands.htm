Interpreter commands

The developer will need the following to write procedures, which call the devices through the interpreter.

The syntax is NOT case sensitive.

The names of devices - three characters beginning with letter (e.g. PRL, CT0)

without reserved names 'Snh' ; 'Itr' and ‘SP#’

  • R : floating point number
  • N : integer number
  • LN : long integer
  • B : boolean
  • Filename, mode, Text : string
  • # : * or alphanumerical symbol (in 1..9,A..Z order)

if Filename does not include a path then configuration path (application path + \User\Config\) is assumed

if Filename is empty then the "Defaults.<DeviceName>" device name is assumed.

 

Remote use:

In case of success the executed command returns either 'OK' or measured/set value , otherwise it returns 'Error:' with optional error description.
